# v1.2.0

## Highlights

- Adds LaTeX-inspired authoring coverage across document geometry, appendix
  numbering, generated lists, tables, figures, equations, chemistry notation,
  references, citations, footnotes, overlays, review annotations, presets, and
  domain-specific namespaces.
- Introduces page layout primitives such as `PageLayout`, orientation-aware
  section geometry, and template-level layout defaults so page size, margins,
  and landscape sections can be configured in one place.
- Refines the public API boundary: common authoring names stay at top level,
  domain-specific objects move behind focused namespaces, and style/default
  configuration objects are canonical under `oodocs.styles`.
- Reworks API reference generation so the user-facing reference no longer
  exposes source paths, coverage tables, or uncategorized API inventory by
  default. Evidence output remains available through the dedicated apidoc
  evidence profile and sidecars.
- Splits the PyPI README from the repository README and trims release assets so
  GitHub Releases attach only the files users need: distributions, the user
  guide PDF, and the API reference PDF.
- Simplifies maintenance by adding pytest markers, shrinking the public API
  naming contract, marking expensive render/example/self-apidoc tests, and
  merging scattered apidoc documentation into focused user, reference, and
  development pages.

## Compatibility Notes

- This release continues the public API cleanup started in v1.1.0. Advanced
  domain features such as algorithms, theorem-like blocks, page positioning,
  PDF page insertion, importers, workflow helpers, and review annotations should
  be imported from their domain namespaces.
- API reference builds should use the base `[tool.oodocs.apidoc]` table for
  user-facing output and `[tool.oodocs.apidoc.evidence]` for coverage or
  inventory evidence.
- Parser extension helpers are canonical under `oodocs.apidoc.docstring`, and
  example validation helpers are canonical under `oodocs.apidoc.examples`.
- Style/default configuration helpers such as `FootnoteDefaults`,
  `HeaderFooterDefaults`, `HeadingStyle`, and `ListStyle` are canonical under
  `oodocs.styles`; top-level compatibility attributes may exist, but they are
  not part of `oodocs.__all__`.

## Release Notes

- Adds documentation and tests for LaTeX package gaps including `geometry`,
  `hyperref`, `cleveref`, `biblatex`/BibTeX, `mhchem`, `amsmath`, `tcolorbox`,
  `enumitem`, `glossaries`, `fancyhdr`, `pdfpages`, `subcaption`, `graphicx`,
  `booktabs`, `longtable`, `tabularx`, algorithm/listing blocks, templates,
  and review annotations.
- Adds `README-PYPI.md` as the package long description and keeps the repository
  README focused on contributor, example, and release workflow context.
- Adds `docs/how-to/compose-api-objects.md`, `docs/reference/apidoc.md`, and
  `docs/development/api-coverage.md` as the consolidated apidoc documentation
  entry points.
- Adds `docs/development/testing.md` with the normal development test command:
  `pytest -m "not slow and not render and not apidoc_full and not examples"`.
- Adds pytest markers for `slow`, `render`, `apidoc_full`, `examples`, and
  `contracts`, and moves the public API contract to `tests/contracts/`.
- Keeps `release-notes/v1.1.0.md` unchanged after the v1.1.0 tag.
- The local fallback version metadata is aligned with the 1.2.0 release line
  for source trees without git tag metadata.
