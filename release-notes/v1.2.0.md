# v1.2.0

v1.2.0 is a broad authoring, API-boundary, documentation, and release
packaging release. The main theme is LaTeX-package-style document authoring:
common LaTeX capabilities now have explicit OODocs APIs, documentation, tests,
examples, or clear renderer-degradation notes.

This note is intentionally detailed so users can search the release page for a
specific feature, class, helper, or import namespace.

## Highlights

- Adds page geometry support with `PageLayout`, `PageSize`, `PageMargins`,
  orientation helpers, and section-scoped layouts.
- Adds a large set of LaTeX-inspired authoring features across generated
  lists, tables, figures, equations, chemistry notation, references, citations,
  footnotes, overlays, review annotations, templates, and domain namespaces.
- Tightens the public API boundary: common authoring names remain convenient at
  top level, while advanced or domain-specific APIs move behind focused
  namespaces such as `oodocs.media`, `oodocs.pdf`, `oodocs.generated`,
  `oodocs.review`, `oodocs.chemistry`, `oodocs.equations`, and
  `oodocs.styles`.
- Reworks user-facing API reference generation so official help output no
  longer exposes absolute source paths, coverage tables, or uncategorized API
  inventories by default. Evidence output remains available through explicit
  apidoc evidence configuration.
- Splits the PyPI long description into `README-PYPI.md` and trims release
  assets so GitHub Releases publish user-facing artifacts instead of every
  development/evidence/example file.
- Adds focused pytest markers and testing documentation so day-to-day checks can
  skip expensive render, examples, and full self-apidoc jobs.

## Searchable Feature Index

### Page Geometry, Headings, and Document Structure

- Adds `PageLayout` as the central page-geometry object for page size, margins,
  and orientation.
- Adds `PageLayout.portrait(...)` and `PageLayout.landscape(...)` helpers.
- Supports `DocumentSettings(page_layout=...)` as the preferred one-place page
  geometry configuration.
- Keeps `DocumentSettings(page_size=...)` and
  `DocumentSettings(page_margins=...)` compatibility paths while documenting
  `page_layout` as the clearer API.
- Adds section-scoped layouts through `Section(..., page_layout=...)` so a
  document can switch page geometry around a section break.
- Adds renderer behavior for landscape sections in DOCX/PDF and HTML
  degradation through page-break/layout regions.
- Documents geometry-package equivalents for page size, margins, text block
  width, portrait/landscape pages, and per-section geometry.
- Adds heading style controls and generated-list level styling through
  `HeadingStyle`, `HeadingNumbering`, and `TableOfContentsLevelStyle`.
- Adds appendix numbering containers so appendix sections can use appendix-style
  numbering without changing ordinary chapter numbering.
- Adds `TitleMatter` settings grouping and documented resolved-author behavior.
- Adds metadata keyword helpers on `DocumentMetadata`.

### Templates and Presets

- Adds document-class-style template presets for common document families.
- Documents template classes such as `JournalArticleTemplate`,
  `TechnicalReportTemplate`, `SoftwareManualTemplate`, and `BookTemplate`.
- Adds a cover page preset and renames cover-page page items to overlays.
- Moves template and preset APIs behind `oodocs.presets` where appropriate.
- Documents preset namespace boundaries so user templates are separated from
  core document authoring APIs.

### Generated Lists and Index Pages

- Adds scoped generated lists so generated pages can be document-wide,
  chapter-scoped, or section-scoped where supported.
- Adds and documents `TableOfContents`, `ListOfTables`, `ListOfFigures`, and
  `ListOfReferences` usage in the new generated-list model.
- Adds `ListOfAlgorithms` for algorithm blocks.
- Adds `ListOfListings` for code listings.
- Adds `ListOfFootnotes` for generated footnote collection pages.
- Adds `ListOfComments` for review comments, TODOs, and margin notes.
- Adds glossary/acronym generated-list support through glossary list pages.
- Documents renderer support and degradation differences for generated lists.

### Tables, Long Tables, and Tabular Layout

- Adds booktabs-style table formatting with `TableStyle.booktabs()`.
- Adds longtable-style continuation metadata for tables that continue across
  pages.
- Adds flexible column specification with `ColumnSpec(width=...)`,
  `ColumnSpec(flex=...)`, `ColumnSpec(key=...)`, `ColumnSpec(header=...)`,
  `ColumnSpec(visible=False)`, and column-level style controls.
- Adds tabularx/array-style table layout documentation for fixed-width and
  flexible-width columns.
- Adds grouped table header helpers.
- Adds row and column span support for `TableCell(rowspan=...)` and
  `TableCell(colspan=...)`.
- Adds `SubTable` and `SubTableGroup` for captioned groups of related tables.
- Adds subtable reference labels and anchor support.
- Adds a table overflow policy with warnings that point users toward
  `ColumnSpec(flex=...)`, `Table.excerpt(...)`, or `Table.save_csv(...)`.
- Exposes table loader layout options for record-based tables.
- Hides low-level table renderer helpers and table coercion helpers from the
  public top-level API.

### Figures, Images, Subfigures, and PDF Pages

- Adds graphicx-style figure transforms and documents image sizing,
  scaling, rotation, and renderer limitations.
- Adds `SubFigure` and `SubFigureGroup` for captioned groups of related
  figures.
- Adds subfigure reference labels and anchor support.
- Moves advanced media helpers behind `oodocs.media`.
- Adds PDF page insertion through `PdfPages(...)` in the `oodocs.pdf`
  namespace.
- Restricts `PdfPages` to the PDF-focused namespace and removes it from generic
  media exports.
- Adds user-facing guidance for `PdfPages("appendix.pdf", pages=[1, 3])`.
- Documents table/media API policy so broad media primitives remain separate
  from core top-level authoring names.

### Page Overlays and Positioning

- Adds scoped page items for document, section, and page-level overlays.
- Adds overlay examples for page items.
- Adds `Shape`, `TextBox`, and `ImageBox` under `oodocs.positioning`.
- Adds `PageItemScope` and documents page-overlay renderer support.
- Moves positioning helpers out of the top-level API.

### Math and Equations

- Adds amsmath-style equation blocks and documented equation subclasses.
- Adds canonical equation factory helpers such as `Equation.aligned(...)` and
  `Equation.cases(...)`.
- Adds optional SymPy support through `Equation.from_sympy(...)`.
- Moves advanced equation classes into `oodocs.equations`.
- Keeps `inline_math(...)` as the canonical inline math helper.
- Removes the old inline math compatibility alias.
- Documents renderer support and fallback behavior for math parsing.

### Chemistry

- Adds mhchem-style chemistry notation.
- Adds inline chemical formulas through `chemical_formula(...)` and
  `ChemicalFormula(...)`.
- Adds `ce(...)` as a chemistry shorthand helper.
- Adds displayed chemical reactions through `ReactionEquation(...)`.
- Moves chemistry helpers behind `oodocs.chemistry`.
- Documents renderer support for chemical formula subscripts, superscripts, and
  reaction-equation display.

### References, Links, URLs, and Citations

- Adds hyperref-style document metadata and link defaults.
- Adds breakable URL support for long links.
- Adds cleveref-style reference helpers including `ref(...)`, `refs(...)`, and
  `ref_range(...)`.
- Adds bracketed reference helpers such as `bracket_ref(...)` and related
  advanced reference formatting helpers.
- Documents object references, page references, indexing helpers, and
  RenderIndex reference helpers.
- Adds BibTeX citation support stabilization and reference-list policy updates.
- Documents citation helpers such as `cite(...)`, citation sources, citation
  defaults, and `ListOfReferences`.
- Removes the citation reference alias that duplicated the canonical reference
  API.
- Guards link helper examples against stale argument ordering.

### Lists, Glossaries, Acronyms, and Locale Defaults

- Adds enumitem-style list controls for bullets, numbered lists, counter styles,
  marker formats, and list spacing.
- Documents `BulletList`, `NumberedList`, `CounterStyle`, and `ListStyle`
  behavior.
- Adds glossary and acronym support through glossary registry helpers, glossary
  terms, acronyms, and glossary list rendering.
- Moves glossary helpers behind `oodocs.glossary` where appropriate.
- Adds locale defaults through `LocaleDefaults` and documents
  `Theme.from_locale(...)` behavior.
- Adds URL, glossary, list, and locale LaTeX mapping documentation.

### Boxes, Callouts, Code Listings, and Algorithms

- Adds tcolorbox-style box options for `Box` and callout-style blocks.
- Adds registered callout variants so custom callout styles can be reused.
- Documents `BoxStyle` and algorithm box style behavior.
- Adds listings-style code block options for line numbers, highlighted lines,
  captions, generated listing entries, and renderer fallbacks.
- Adds `ListOfListings` for generated listing pages.
- Adds algorithm blocks through `Algorithm(...)`.
- Adds `ListOfAlgorithms` and documents algorithm/listing renderer behavior.
- Moves engineering helpers behind `oodocs.engineering`.

### Footnotes, Headers, Footers, and Review Annotations

- Adds footnote stream defaults through `FootnoteDefaults` and `FootnoteStyle`.
- Adds generated footnote pages through `ListOfFootnotes`.
- Adds header/footer defaults through `HeaderFooterDefaults`.
- Documents fancyhdr-style header/footer slots, page-number formatting, and
  renderer limitations.
- Adds TODO and margin-note review annotations through `Todo`, `todo(...)`,
  `MarginNote`, and `margin_note(...)`.
- Adds generated comment collection through `ListOfComments`.
- Moves review annotations behind `oodocs.review`.

### Public API Boundary and Namespaces

- Adds and documents a public API tier policy.
- Reduces the top-level export contract to a smaller set of core authoring
  names; `TOP_LEVEL_EXPORT_LIMIT` is now 90.
- Keeps core document authoring names such as `Document`, `DocumentSettings`,
  `DocumentMetadata`, `TitleMatter`, `Chapter`, `Section`, `Paragraph`,
  `Table`, `Figure`, `ImageData`, `Box`, `CodeBlock`, `Equation`, `BulletList`,
  `NumberedList`, `Text`, `bold`, `italic`, `inline_code`, `inline_math`,
  `link`, `cite`, `ref`, `refs`, and `ref_range` convenient.
- Moves domain-specific APIs into focused namespaces:
  `oodocs.engineering`, `oodocs.structure`, `oodocs.positioning`,
  `oodocs.review`, `oodocs.generated`, `oodocs.presets`, `oodocs.pdf`,
  `oodocs.media`, `oodocs.equations`, and `oodocs.chemistry`.
- Moves style/default configuration helpers into `oodocs.styles`, including
  `FootnoteDefaults`, `FootnoteStyle`, `HeaderFooterDefaults`,
  `HeadingNumbering`, `HeadingStyle`, `LinkDefaults`, `LocaleDefaults`,
  `ListStyle`, and `GeneratedContentDefaults`.
- Hides internal renderer hooks, normalization helpers, importer helpers,
  generated helpers, media coercion helpers, positioning coercion helpers, and
  stale compatibility aliases from user-facing API inventories.
- Adds tests that prevent stale names from returning to generated API docs.

### API Reference, API Coverage, and PyPI Description

- Changes `ApiPresentationProfile.help()` so user-facing help output does not
  include source paths by default.
- Keeps source paths available only through explicit source-aware profiles such
  as `ApiPresentationProfile.evidence()` or
  `ApiPresentationProfile.help().with_source()`.
- Changes `ApiHelpBookConfig.include_uncategorized_appendix` default to
  `False` for user-facing help books.
- Keeps uncategorized API inventory available for evidence/coverage workflows
  through explicit config.
- Adds `check_api_help_categories(...)` so CI can fail category coverage gaps
  without appending a public "uncategorized API" dump to the user reference.
- Splits apidoc configuration between the base `[tool.oodocs.apidoc]` user
  reference settings and `[tool.oodocs.apidoc.evidence]` coverage/evidence
  settings.
- Moves parser registry and extension helpers to `oodocs.apidoc.docstring`.
- Moves doctest/example helpers to `oodocs.apidoc.examples`.
- Keeps top-level `oodocs.apidoc` focused on ordinary user APIs such as
  `collect_api`, `ApiPackage`, `ApiModule`, `ApiObject`, `ApiParameter`,
  `ApiCollectConfig`, `ApiHelpBookConfig`, `ApiPresentationProfile`,
  `check_api_docs`, and `diff_api`.
- Sanitizes API source paths consistently so generated docs and sidecars avoid
  local absolute paths.
- Separates user API reference output from coverage/evidence output.
- Uses sidecars such as `api_object_tree_json`, `api_coverage_json`, and
  `api_coverage_csv` for machine-readable evidence.
- Adds `README-PYPI.md` so PyPI no longer receives contributor/release workflow
  material from the repository README.

### Release Assets and Examples

- Trims GitHub Release assets to user-facing files: distributions, the usage
  guide PDF, and the API reference PDF.
- Stops publishing duplicate DOCX/HTML API reference files when the PDF is the
  official user-facing reference artifact.
- Keeps broad examples and evidence outputs available in the repository and CI,
  but avoids shipping every generated example as a release asset.
- Adds a purpose-based example catalog and normalizes standalone example CLI
  entry points.
- Adds or refreshes example material for configuration references, CLI manuals,
  project metadata reports, validation gate reports, conformance matrices,
  engineering reports, page overlays, template catalogs, custom styles, and API
  object composition.

### Testing and Maintenance

- Adds pytest markers for `slow`, `render`, `apidoc_full`, `examples`, and
  `contracts`.
- Moves the public API naming/contract checks into `tests/contracts/`.
- Documents the normal development command:
  `pytest -m "not slow and not render and not apidoc_full and not examples"`.
- Adds release evidence sidecars for validation diagnostics where useful.
- Guards tracked docs and generated sidecars against local absolute paths.
- Adds cleanup completion criteria tests that keep documented import locations,
  examples, and support matrices aligned with the code.

## Compatibility Notes

- This release continues the public API cleanup started in v1.1.0. Advanced
  features that used to be visible near the top level may now need domain
  imports, for example `from oodocs.media import ColumnSpec`,
  `from oodocs.pdf import PdfPages`, `from oodocs.generated import
  ListOfListings`, `from oodocs.review import MarginNote`, or
  `from oodocs.chemistry import ReactionEquation`.
- Use `DocumentSettings(page_layout=PageLayout(...))` for new page-geometry
  code. Older page size and margin fields remain understood, but `PageLayout`
  is the clearer API.
- Use `ApiPresentationProfile.help()` for user-facing API help and
  `ApiPresentationProfile.evidence()` for source-path, coverage, and inventory
  evidence output.
- Use the base `[tool.oodocs.apidoc]` table for official user reference output
  and `[tool.oodocs.apidoc.evidence]` for coverage/evidence output.
- Parser extension helpers are canonical under `oodocs.apidoc.docstring`, and
  example validation helpers are canonical under `oodocs.apidoc.examples`.
- Style/default configuration helpers are canonical under `oodocs.styles`; they
  may still be importable through compatibility paths, but they are no longer
  part of the preferred top-level `oodocs.__all__` surface.
- The v1.2.0 release bundle is intentionally smaller than older bundles. Large
  example/evidence outputs should be generated from the repository when needed
  instead of being treated as user release assets.
