# v1.1.0

v1.1.0 is the API documentation and public API cleanup release. It introduced
the composable `oodocs.apidoc` system, rendered self API references, API
coverage/diff evidence, and a large naming cleanup across public authoring,
workflow, style, import, renderer, and result APIs.

This note was expanded after publication to make the already-released changes
searchable by feature and API name. The v1.1.0 tag and release artifacts are not
rebuilt by this note update.

## Highlights

- Completes a broad public API documentation pass with Google-style docstrings,
  examples, parameter/return/raises/yields sections, renderer notes, and
  standardized result object documentation.
- Adds the composable API documentation object model:
  `ApiPackage`, `ApiModule`, `ApiObject`, `ApiParameter`, `ApiReturn`,
  `ApiException`, `ApiExample`, `ApiSeeAlso`, `ApiRendererNote`,
  `ApiDocIssue`, `ApiCoverageResult`, `ApiDiffResult`, and `ApiSnapshot`.
- Adds API collection from live modules, source trees, package roots, and
  pyproject-configured targets through `collect_api(...)`,
  `collect_module_api(...)`, `collect_object_api(...)`, `ApiCollectConfig`,
  and `ApiPublicPolicy`.
- Adds API help-book rendering through `ApiHelpBookConfig`, `to_help_book(...)`,
  `to_chapter(...)`, `to_section(...)`, `to_blocks(...)`,
  `to_summary_table(...)`, `save_all(...)`, and sidecar outputs.
- Adds presentation profiles with `ApiPresentationProfile.reference()`,
  `help()`, `compact()`, `manual()`, `evidence()`, `review()`, and
  `website()`.
- Adds API coverage, snapshot, and diff workflows through `check_api_docs(...)`,
  `diff_api(...)`, API coverage sidecars, API diff sidecars, and CLI support.
- Cleans up public naming across apidoc, workflow, import, result, style/theme,
  validation, renderer, table, media, generated-page, and inline helper APIs.
- Moves visual styling into the `oodocs.styles` namespace with grouped theme
  defaults, named stylesheet entries, reusable padding/border/counter
  primitives, and renderer validation for unknown style names.

## Searchable Feature Index

### API Object Model

- Adds `ApiPackage` as the root API inventory object.
- Adds `ApiModule` for module-level API pages and module summaries.
- Adds `ApiObject` for functions, classes, methods, properties, attributes, and
  values.
- Adds `ApiParameter`, `ApiReturn`, `ApiException`, `ApiExample`,
  `ApiSeeAlso`, and `ApiRendererNote` as reusable API documentation records.
- Adds `ApiDocIssue`, `ApiCoverageResult`, `ApiCoverageIssue`,
  `ApiDiffResult`, and `ApiSnapshot` for review/evidence workflows.
- Adds API object query helpers, package tree helpers, leaf metadata helpers,
  value helpers, renderer note helpers, and block composition helpers.
- Adds direct composition of API objects into ordinary OODocs documents, so an
  API object can become a section, block list, summary row, or full help-book
  page.

### API Collection

- Adds `collect_api(...)` for package/repository API collection.
- Adds `collect_module_api(...)` and `collect_object_api(...)` facades for
  focused collection.
- Adds `ApiCollectConfig` and pyproject loading for reusable collection config.
- Adds `ApiPublicPolicy` with `__all__`, underscore, all, and explicit public
  boundary modes.
- Adds griffe-backed collection for source analysis.
- Adds inspect fallback collection for live objects and environments where
  source collection is unavailable.
- Adds support for inherited members, imported aliases, reexported class member
  docs, constructor docstring merging, overload metadata, private-member toggles,
  source-location toggles, and member-include toggles.
- Adds target-aware collection from repo roots, package directories, single
  files, namespace packages, setuptools package dirs, py-modules, Hatch, Poetry,
  PDM, Flit, import-names, and only-include style layouts.
- Adds diagnostics for griffe fallback and inspect collector behavior.

### Docstring Parsing and Parser Extensions

- Adds reusable docstring parser objects and parser registry support.
- Supports Google, NumPy, Sphinx, and Markdown-style docstring sections.
- Parses parameters, keyword arguments, returns, raises, yields, deprecation
  notes, examples, see-also/related API directives, Sphinx varargs fields, and
  extended parameter sections.
- Adds custom parser loading from config paths and parser import modules.
- Adds parser config roundtrip tests and parser sidecar serialization.
- Adds `parse_docstring(...)`, parser module loading, parser name listing, and
  parser registration APIs.

### API Help Books and Rendering

- Adds `ApiHelpBookConfig` as the named build configuration for rendered API
  help books.
- Adds `output_formats`, `output_dir`, `max_heading_level`, and
  `presentation=...` naming.
- Adds rendering to DOCX, PDF, HTML, JSON sidecars, and CSV sidecars.
- Adds category-first help pages and symbol-role organization.
- Adds compact syntax sections, wrapped signatures, signature truncation by
  profile, compact see-also links, unnumbered API tables, and value-definition
  rendering.
- Adds relative source path rendering for profiles that include source
  information.
- Adds profile-specific rendering for reference, help, compact, manual,
  evidence, review, and website use cases.
- Adds website profile links to stable anchors.
- Adds review notes to apidoc profiles.
- Adds `ApiHelpBookConfig.from_pyproject(".").save_all(".")` style workflows.

### API Coverage, Checks, Snapshots, and Diffs

- Adds `check_api_docs(...)` and CLI coverage checks.
- Adds API coverage thresholds, coverage deltas, issue tables, missing docs,
  missing examples, doctest coverage counts, annotation changes, and executable
  doctest coverage checks.
- Adds `diff_api(...)` and rendered diff bundles.
- Adds API snapshot sidecars and readback helpers.
- Adds sidecar keys such as `api_object_tree_json`, `api_coverage_json`, and
  `api_coverage_csv`.
- Renames the object tree sidecar suffix to `{stem}-object-tree.json`.
- Adds coverage/diff evidence rendering for mixed repositories and general
  package layouts.
- Adds full self-reference API rendering in release workflows.

### API Examples and Self Documentation

- Adds `examples/api_objects_example/` as a working example for collecting API
  objects, rendering help books, composing API objects into documents, writing
  object tree JSON, and writing coverage sidecars.
- Adds general repository API object examples, single module examples, package
  variant examples, custom parser examples, griffe mixed-repo examples, and
  manual composition examples.
- Adds release sidecars for API object trees and API coverage.
- Adds tests that render composed apidoc bundles across formats.
- Adds tests that assert self API reference output quality.

### Public API Naming Cleanup

- Replaces `ApiDocProfile`, `profile=...`, and `oodocs.apidoc.styles` with
  `ApiPresentationProfile`, `presentation=...`, and
  `oodocs.apidoc.profiles`.
- Replaces the old apidoc build naming with help-book naming:
  `ApiHelpBookConfig`, `output_formats`, `output_dir`, and
  `max_heading_level`.
- Renames apidoc selection, row helper, composition helper, display text,
  exception, renderer note, JSON file, and CLI option APIs to consistent names.
- Removes the old apidoc build CLI after routing through the new config-driven
  API.
- Adds public API surface inventory and public API naming guards so stale names
  do not return accidentally.
- Adds API reference scope documentation and explicit public API boundary
  tests.

### Inline, Component, and Authoring Naming Cleanup

- Renames old inline helpers to canonical names such as `inline_code(...)`,
  `text_color(...)`, `highlight_color(...)`, `inline_math(...)`, and
  `strikethrough(...)`.
- Renames paragraph, textbox, table, caption, title, block alignment, image DPI,
  image format, page numbering, heading counter, citation style, text style,
  generated content, and list marker fields to clearer public names.
- Renames the fourth-level section class to the canonical fourth-level section
  naming.
- Renames generated content classes and generated content defaults.
- Hides table renderer helper methods from the public API.

### Result, Workflow, Import, and Adapter APIs

- Adds common result helpers for JSON, CSV, text, table, and issue-row output.
- Standardizes workflow output APIs around `OutputBundle`, `outputs`,
  `source_type`, `document_factory`, `fail_on_warning`, and `report_format`.
- Updates workflow CLI options to match Python API naming.
- Makes `Document.save_all(...)` return `OutputBundle`.
- Renames apidoc file IO APIs to `load_*` and `save_*` style names.
- Makes Markdown and notebook import parsers return structured `ImportResult`
  objects; use `.blocks` when a block list is needed.
- Renames notebook import options, import policy values, and import issue line
  fields.
- Adds object-centered adapter APIs and renames evidence/record table missing
  policy options.

### Style and Theme System Cleanup

- Moves style classes into focused style modules and the `oodocs.styles`
  namespace.
- Adds grouped theme defaults instead of flat theme fields.
- Adds `Theme`, `StyleSheet`, `TextStyle`, `ParagraphStyle`, `TableStyle`,
  `TableCellStyle`, `BoxStyle`, `Padding`, border primitives, stroke
  primitives, counter styles, chip styles, generated-list level styles, and
  named style registration/resolution.
- Adds stylesheet CSS class hooks for HTML output.
- Moves chip and table-cell styles into the style namespace.
- Moves presets to named stylesheet styles.
- Adds theme default resolvers and grouped default validation.
- Guards apidoc presentation profiles from being confused with visual styles.

### Release Rendering and Assets

- Expands release rendering so the workflow can build the user guide, curated
  example documents, API reference bundles, API sidecars, template PDF, and
  evidence report.
- Renders all curated examples in the release workflow.
- Publishes API reference sidecars for object trees and coverage evidence.
- Aligns local fallback version metadata with the 1.1.0 release line for source
  trees without git tag metadata.

## Compatibility Notes

- v1.1.0 intentionally removes old aliases as part of the API cleanup. It is not
  a backwards-compatible alias-preservation release.
- Use `ApiPresentationProfile`, `presentation=...`, and
  `oodocs.apidoc.profiles` instead of `ApiDocProfile`, `profile=...`, and
  `oodocs.apidoc.styles`.
- Use `ApiHelpBookConfig`, `output_formats`, `output_dir`, and
  `max_heading_level` instead of older apidoc build/config names.
- Use `api_object_tree_json`, `api_coverage_json`, and `api_coverage_csv` for
  API reference sidecar outputs.
- Use `inline_code(...)`, `text_color(...)`, `highlight_color(...)`,
  `inline_math(...)`, and `strikethrough(...)` instead of old inline helper
  aliases.
- Use `.blocks` from `ImportResult` when Markdown or notebook import output is
  needed as a plain block list.
- Use grouped theme defaults, `Theme`, `StyleSheet`, and named styles instead
  of flat theme fields or renderer hook helpers.
- Use `OutputBundle` and the standardized workflow naming for automation and
  CLI integrations.
