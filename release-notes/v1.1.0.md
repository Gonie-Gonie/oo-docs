# v1.1.0

## Highlights

- Completes the public API documentation pass with Google-style docstrings,
  richer examples, and standardized sections for user-facing classes,
  functions, methods, and result objects.
- Reworks `oodocs.apidoc` around composable API objects and help-book output:
  collectors produce `ApiPackage` trees, selected symbols can be inserted into
  ordinary OODocs documents, and full API reference bundles can be rendered to
  DOCX, PDF, HTML, and sidecar files.
- Adds MATLAB-style API help pages with category-first navigation, compact
  syntax sections, relative source paths, unnumbered API tables, compact
  see-also links, and coverage/evidence appendices.
- Cleans up public naming across apidoc, style/theme, workflow, import,
  adapter, generated-page, table, media, result, and validation APIs.
- Moves visual styling to the `oodocs.styles` namespace with grouped theme
  defaults, named stylesheet entries, reusable padding/border/counter
  primitives, and renderer validation for unknown style names.
- Narrows GitHub Release assets to the package distributions plus the
  user-facing user guide and API reference PDFs, leaving example outputs,
  machine-readable sidecars, DOCX variants, and HTML variants out of the
  release download list.

## Compatibility Notes

- This release intentionally removes old aliases and does not preserve
  backwards compatibility for the API cleanup.
- Use `ApiPresentationProfile`, `presentation=...`, and
  `oodocs.apidoc.profiles` instead of `ApiDocProfile`, `profile=...`, and
  `oodocs.apidoc.styles`.
- Use `ApiHelpBookConfig`, `output_formats`, `output_dir`, and
  `max_heading_level` for API reference bundle configuration.
- API reference sidecar outputs now use `api_object_tree_json`,
  `api_coverage_json`, and `api_coverage_csv`. The object tree file uses the
  `{stem}-object-tree.json` suffix.
- Use `inline_code`, `text_color`, and `strikethrough` instead of the old
  inline helper aliases.
- Markdown and notebook parser APIs now return structured `ImportResult`
  objects. Use `.blocks` when a block list is needed.
- Workflow output APIs use `OutputBundle`, `outputs`, `source_type`,
  `document_factory`, and `fail_on_warning` naming across Python and CLI
  surfaces.
- Visual theme customization should go through grouped defaults, `Theme`,
  `StyleSheet`, and named styles rather than flat theme fields or renderer
  helper hooks.

## Release Notes

- The public API surface is guarded by snapshot and naming tests so removed
  names such as `ApiDocProfile`, old row helpers, old CLI flags, and flat theme
  aliases do not return accidentally.
- API documentation builds can now be configured from `pyproject.toml` and
  rendered through `ApiHelpBookConfig.from_pyproject(".").save_all(".")`.
- The `examples/api_objects_example/` bundle now demonstrates help-book
  rendering, object composition, API object tree JSON, and coverage sidecars.
- Release assets now include the Python distributions, the usage guide PDF, and
  the API reference PDF. Example outputs, API sidecars, DOCX variants, and HTML
  variants remain reproducible from the repository but are not attached to the
  GitHub Release.
- The local fallback version metadata is aligned with the 1.1.0 release line
  for source trees without git tag metadata.
