# v2.0.0

v2.0.0 completes the generic documentation-model cleanup. It replaces
application-specific defaults and compatibility paths with explicit,
renderer-neutral objects shared by DOCX, PDF, and HTML.

## Added

- Generic `CoverPage` content and reusable accented/centered cover presets.
- Explicit `FrontMatter`, `MainMatter`, and `BackMatter` containers with
  front/main/back page-number policy.
- Plain `ObjectLink` navigation alongside numbered, locale-aware reference
  templates.
- Semantic `DescriptionList` blocks, schema and CLI documentation models, and
  optional external collectors under `oodocs.integrations`.
- Per-line aligned-equation anchors and references, scoped numbering policies,
  presentation-only `Quantity` values, and optional Pint/SymPy integrations.
- Lossless BibTeX field retention, optional parser backends, richer citation
  formatting, and field-loss diagnostics.
- `DocumentSuite` for documents that share explicit variables, citations,
  assets, and output policy.
- A compact generic manual fixture and cross-renderer semantics contract.

## Changed

- External parsers and collectors now live under `oodocs.integrations`; core
  models accept ordinary Python mappings and sequences without optional
  dependencies.
- Reference labels and their placement are controlled by locale/templates
  instead of renderer literals.
- Table/dataframe, long-table, wide-table, grouped-header, and validation
  contracts are documented and tested against the existing `Table` API.
- The core top-level API now includes `CoverPage`, the three matter containers,
  `DescriptionItem`, and `DescriptionList`. Domain models remain in focused
  namespaces.

## Removed

- The `oodocs.adapters` namespace and its compatibility imports.
- Project-branded cover factories, release-evidence defaults, fixed evidence
  file skeletons, and other application-owned names/defaults.
- The legacy `Equation.from_sympy` entry point. Import
  `equation_from_sympy` from `oodocs.integrations.sympy` instead.

## Migration

- Applications should pin `oodocs>=2.0.0` before adopting the new generic
  models. The EPlusSimple migration checklist uses this as its minimum OODocs
  version.
- Import schema, CLI, engineering, evidence, suite, and integration APIs from
  their documented namespaces; do not copy application models into OODocs.
- Replace numbered references to unnumbered targets with `target.link(...)`.
- Replace inferred front matter with explicit matter containers when page
  numbering or generated front/back pages must be stable.
